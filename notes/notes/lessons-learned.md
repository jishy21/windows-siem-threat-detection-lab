# Lessons Learned

## Overview

This lab provided hands-on experience building a Windows-based SIEM environment using Splunk Enterprise, Splunk Universal Forwarder, and Sysmon.

The project reinforced foundational concepts related to security monitoring, endpoint visibility, Windows logging, and incident investigation.

## Key Skills Learned

- Installing and configuring Splunk Enterprise
- Configuring Splunk Universal Forwarder
- Collecting Windows Security, System, and Application logs
- Installing and using Sysmon for enhanced logging
- Creating Splunk detection queries
- Investigating Windows security events
- Monitoring PowerShell activity
- Detecting failed logins and suspicious account activity
- Writing incident response documentation
- Organizing technical documentation in GitHub

## Security Concepts Reinforced

- Centralized logging
- SIEM monitoring
- Event correlation
- Authentication monitoring
- Privilege escalation detection
- Endpoint visibility
- Least privilege awareness
- Incident investigation workflows

## Challenges Encountered

- Troubleshooting log forwarding between the client and Splunk server
- Verifying the correct Windows Event IDs
- Configuring Sysmon operational log collection
- Understanding how Splunk indexes and searches events
- Identifying the correct detection queries for each activity

## Improvements for Future Versions

- Add Active Directory integration
- Forward logs from multiple endpoints
- Create custom Splunk dashboards
- Configure automated email alerts
- Add brute-force detection thresholds
- Integrate Linux log monitoring
- Add threat intelligence feeds

## Final Thoughts

This project helped simulate how enterprise environments monitor endpoints and investigate suspicious activity using centralized logging solutions. It also improved troubleshooting, documentation, and security analysis skills that are valuable for help desk, SOC analyst, and entry-level cybersecurity roles.
